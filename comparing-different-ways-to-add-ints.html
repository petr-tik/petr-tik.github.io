Comparing different ways to add ints

// under algorithms hackerrank python

Puzzle

Given 2 ints represented as array of digits in order (eg. 843 = [8, 4, 3]), implement a function that returns a sum of the 2 ints in the same format.

eg. [8, 4, 3] + [1, 8, 2] = [1, 0, 2, 5]

Idea

Compare recursive and iterative solution. Time them and see how long each runs on the same input.

Bug

For every iteration, I checked if the result was the same. I'd been running arrays of length > 500 before I realised the iterative and recursive solutions diverged. When inspected, the recursive solution returned an array much longer than expected. This was the wrapper's fault, as was proven by running the methods without wrapping.

My hypothesis was: The timeit wrapper method tooks the args and kwargs of the wrappee and kept them across runs. From the second interation onwards, the res array in the wrapped recur_sum_helper was kept inside the wrapper. Adding 

    if not res:
        res = list()

solved the problem.
